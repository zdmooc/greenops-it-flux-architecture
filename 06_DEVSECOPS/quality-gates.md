# Quality Gates

## Principes

- Ne pas bloquer brutalement tout le legacy.
- Commencer par la visibilité.
- Définir des seuils progressifs.
- Prioriser les vulnérabilités critiques.

## Indicateurs

- dette critique restante ;
- nouveaux bugs critiques ;
- vulnérabilités critiques ;
- couverture tests ;
- délai moyen de correction ;
- exceptions ouvertes.
